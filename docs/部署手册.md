# 个人网站部署手册

## 最小原理

- 网页文件：HTML、CSS、JavaScript。
- 托管平台：把文件放到公网可访问的位置。
- 域名：把易记地址指向托管平台。
- HTTPS：让浏览器信任网站，也是 PWA / Service Worker 的前提。

本项目第一版只输出静态文件，所以不需要服务器、后端、数据库。

## 本地构建

```powershell
npm install
npm run build
```

构建完成后，上传或部署 `dist/`。

如果发布到 GitHub Pages 项目页，使用：

```powershell
npm run build:pages
```

这会把静态资源路径适配为 `/personal-website/`。

## 最快上线：Vercel

1. 把本目录提交到 GitHub 仓库。
2. 打开 Vercel，选择导入 GitHub 项目。
3. Build Command 使用 `npm run build`。
4. Output Directory 使用 `dist`。
5. 部署完成后，先用 Vercel 默认 HTTPS 地址访问。
6. 用手机流量和家庭 Wi-Fi 做国内访问实测。

`vercel.json` 已经写入最小配置；如果 Vercel 自动识别 Vite，也可以直接使用默认配置。

## 临时上线：GitHub Pages

当 Vercel 账号验证暂时无法通过时，先用 GitHub Pages 临时发布：

1. 仓库需要公开，GitHub Free 下私有仓库 Pages 发布受限。
2. 运行 `npm run build:pages`。
3. 把 `dist/` 内容发布到 `gh-pages` 分支。
4. 在 GitHub Pages 设置里选择 `gh-pages` 分支和根目录。
5. 访问 `https://amymiller76717-sudo.github.io/personal-website/`。

注意：GitHub Pages 国内访问可能慢或不稳定，只作为临时公开 URL。

当前临时发布状态：

- 线上地址：`https://amymiller76717-sudo.github.io/personal-website/`
- 发布分支：`gh-pages`
- Pages 来源：`gh-pages` 分支根目录
- 仓库可见性：public

## 备选：Cloudflare Pages

1. Workers & Pages 新建 Pages 项目。
2. 连接 GitHub 仓库。
3. Build command 填 `npm run build`。
4. Build output directory 填 `dist`。
5. 部署后先用默认 Pages 地址实测。

Cloudflare Pages 免费额度和静态托管体验好，但中国大陆访问也不能保证稳定。

## 国内友好免费试运行：EdgeOne Makers / Pages

当目标是测试国内网络体验时，优先试 EdgeOne 免费方案。

推荐导入 GitHub 仓库：

- 仓库：`https://github.com/amymiller76717-sudo/personal-website`
- 分支：`main`
- 框架：Vite
- 安装命令：`npm install`
- 构建命令：`npm run build`
- 输出目录：`dist`
- 环境变量：无

如果 GitHub 导入失败，可以直接上传 `dist-edgeone-upload.zip`。

完整步骤见 `docs/EdgeOne免费部署清单.md`。

## 长期国内路线：阿里云 / 腾讯云

国内稳定优先时，最终应迁移到：

- 阿里云 OSS + CDN + HTTPS
- 腾讯云 COS + CDN + HTTPS

注意：

- 中国内地节点加自定义域名通常需要 ICP 备案。
- 暂时不备案时，可以用香港或境外节点过渡，但不能承诺所有国内网络稳定。
- 迁移时不要重写网站，只上传同一个 `dist/`。

## 国内访问实测

每次部署后至少测试：

- 手机 4G/5G 打开。
- 家庭 Wi-Fi 打开。
- 刷新 5 次，看是否偶发失败。
- 首屏是否能在可接受时间内出现。
- 如果以后加入 PWA，再测添加到桌面和离线打开。

记录模板见 `docs/国内访问实测表.md`。

## 迁移原则

- 内容和源码放在 Git，不把平台当作唯一资产。
- 站内链接尽量使用普通相对路径。
- 构建产物固定为 `dist/`。
- 不使用某个平台专属函数，除非以后明确要做后端功能。
